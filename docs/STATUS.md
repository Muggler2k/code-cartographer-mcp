# Project Status

_Last updated: 2026-06-15 · Status: **GA — v1.0.0** (released 2026-06-15 as an internal MCP tool; CAS ADR 0034 §4)_

## TL;DR

**The full product is implemented and tested (Epics A–Q*, incl. O).** (*epic letters follow the ADR 0028 roadmap order — Q, then O, landed after M.) `init_codebase`
builds, persists (atomic + gitignored), and `check_init_state` stale-checks a real
`.code-cartographer-mcp/context-map.json` carrying files, languages, entry points,
modules, ownership signals, a provider-extracted **static call graph**, and
confidence-graded findings. Four provider tiers: the **TS/JS provider** (TS
compiler API) type-resolves cross-file edges (`confirmed`); the **C#/VB Roslyn
provider** (ADR 0027 — an optional .NET sidecar, `confirmed`; without a .NET SDK
C# falls back to tree-sitter; per ADR 0032 it binds through the host SDK's BCL
surface — repo-internal calls flowing through LINQ/framework types resolve, external
targets stay `unresolved#name` — and emits property/field ownership signals that
never become call-graph nodes; per **ADR 0033** it also claims **Visual Basic** —
a separate `VisualBasicCompilation` per batch with the same extraction semantics,
where VB data access via invocation syntax is never an edge, cross-language VB↔C#
calls stay `unresolved#name` (disclosed), and without the SDK VB degrades to the
heuristic floor since no maintained tree-sitter VB grammar exists); the **tree-sitter
provider** parses 8 languages (Python, Go, Java, Rust, Ruby, C#, C++, C) via WASM
grammars (`likely`), with cross-file resolution for Go (packages), Python
(imports), and Rust (`use`/modules); a heuristic regex floor covers the rest
(`candidate`). All **20 MCP tools** and the CLI work end-to-end (dogfooded on this
repo). A **static path-finding subsystem** (bidirectional-BFS fewest-hop, max-bottleneck
best-confidence, k-best, Tarjan SCC) over a **derived `graph-index.sqlite`** (built-in
`node:sqlite`) backs indexed caller/callee/path queries (ADR 0023). Internal seams are
unified per **ADR 0025**: every graph query runs through one analysis-context envelope
(`src/analysisContext.ts`, injectable for tests), the MCP + CLI surfaces are two adapters
over one declarative tool table (`src/tools.ts`), and the type vocabulary lives in
`src/schema.ts` apart from the map engine. Findings derivation v2 (**ADR 0026**) adds
seven map-only rules — cycles, visibility hotspots, source→test violations, scattered
ownership, statically untested modules, fan-out hotspots, entry-point orphans — plus
re-export visibility (`OwnershipSignal.reExport`; barrels stop false-positiving the
duplicate rule). A **capability evaluation harness** (ADR 0029, Epic M — the first step of the **v0.2 "Verified Static Context"** roadmap, ADR 0028) scores the analyzer against golden-annotated fixture repos + external real repos: all seven synthetic fixtures — incl. **`cpp-small`**, the C++ tree-sitter-tier baseline brought into the gate as Epic N groundwork (ADR 0035, accepted) — pass their required golden items with zero invariant violations (`npm run eval`; CI-gated; `vb-small` joined with ADR 0033). **Benchmark gates** (ADR 0030, Epic Q) pin the structural metrics — node/edge counts, fixed-query expansion + SQLite query counts, SCC-built-once, output size ±20% — against `eval/baselines.json` (hard, deterministic), with wall-clock under generous sanity ceilings and heap/external numbers record-only. **Diff/PR mode** (ADR 0031, Epic O) ships the review primitive as one tool, **`analyze_diff`** (CLI `diff`): a bounded six-section delta between the persisted baseline map and a fresh in-memory rebuild of the current tree under the baseline's recorded scope (or an explicit `baselineMapPath` snapshot) — changed files, graph node/edge deltas, new/resolved duplicates, legacy reachability transitions, new/resolved risk areas, per-edge confidence regressions (static evidence weakening, never a runtime claim) — plus the `addedParallelPath`/`bypassedAbstraction`/`revivedLegacy`/`increasedUncertainty` verdict and one recommendation. Per **ADR 0034** (production validation — agents-via-Claude-Code; accepted), the S1 token-bounding slice digests every over-budget `llm_readable` payload — `get_context_summary`/`init_codebase` (~536k → ~2.5k tokens on a medium repo), `map_call_stack`, `detect_architecture_drift`, `analyze_reachability`, and the `visualize_call_stack` diagram (rendered from a bounded root-connected subgraph) — each capping sample lists at **8** (the second S1 pass — every tool's `llm_readable` now lands under the ~2000-token agent budget; whole dogfood suite ~46k → ~15k) while preserving counts + true totals + per-field breakdowns + the codebase-only boundary; the four list-digests share one `cappedSample`/`cappedSection`/`digestNote` helper set, and the previously-uncapped recorded-scope `dirs`/`patterns` are bounded the same way. S1's **never-throw half** is also done: the build degrades pathological inputs — unreadable/vanished (a new `"unreadable"` `AnalysisReason`), over-cap, binary, non-UTF8, broken syntax, symlinks (cycle + dangling), submodule gitlinks — to disclosed non-analyzable records instead of throwing, pinned by `test/robustness.test.ts`. **Epic N (C++)** is underway (ADR 0035, header-aware tree-sitter resolver, clang deferred): the `cpp-small` eval baseline gates the tier, and **N-S1 is landed** — C++ methods are now qualified by their enclosing class (`Calculator::run` ≠ `Widget::run`) with a scope-aware intra-file resolver (member hides a same-named free function), `qualifyByType`-gated so other tree-sitter languages are untouched; **N-S2 is landed** too — a bare unresolved call binds to the unique non-static free-function definition in the SAME directory (a directory-scoped proxy, `likely`-clamped; the full `#include`-graph for cross-directory links is deferred); and **N-S3 (namespaces + `using`) is landed** — a function in `namespace ns` is qualified `ns::f` and a scoped `Scope::name()` call binds to the declaration whose node id is the call's full `::`-path (so a deep `outer::inner::deep()` never collides with an unrelated `inner::deep`); `using ns::f;` and `using namespace ns;` resolve bare calls to in-file declarations, with in-file-only / `length===1` / conflicting-alias ambiguity guards that stay `unresolved` rather than guess; and a scoped namespace-member call whose target lives in another same-directory file binds cross-file (`useRemote → lib.cpp#remote::fetch`) — the same-dir `cppGlobals` index extended to namespace members, with class members and out-of-line `int Type::m(){}` defs excluded so member calls never over-resolve; and a same-dir miss binds across DIRECTORIES via the `#include` graph (`parseCppIncludes` + a whole-repo `cppDefsByName` index → the unique include-connected sibling-source definition, `#include "x.h"` → `x.cpp`; the include graph scopes the index so unrelated same-named defs never over-resolve) — the staged tree-sitter C++ resolver is now feature-complete. Exercised by the `npm run dogfood` harness over the real MCP transport. The **S2** slice adds `src/telemetry.ts` — gated, local-file/dev-only, code-content-free telemetry emitted from the MCP handler; the **S3** honesty SLO ("zero false confirmed") is already gated by the eval's exact-confidence edge matching + `checkInvariants`, and the **S4** slice validates ADR 0011 staleness + incremental re-analysis under a churn sequence (`test/churn.test.ts`) and gates conversational **cold-start latency** — fresh-process measurements (`eval/measureColdStart.ts`) checked against per-target SLAs in `eval/baselines.json` (`test/latencySla.test.ts`) with generous CI headroom (ADR 0030's anti-flakiness mandate), covering all the cold-start drivers ADR 0034 names: WASM tree-sitter grammar load (`cpp-namespaces`), TS Program build (`self`, a real ~144-file multi-language repo), **Roslyn sidecar warm-up** (`csharp-small`, dotnet-gated on a pre-built sidecar), count-scaling/O(n²) over a ~5000-file synthetic repo (`large`, CI-only), and a REAL pinned corpus (`node_modules/typescript/lib` — the TS compiler's own lib, byte-identical dev↔CI, a stable real-complexity gate that a user can extend by adding any repo path as a baseline key) — and that same pinned corpus is now **also an honesty-invariant gate** (`test/realCorpusGate.test.ts`, ADR 0034 S7): `checkInvariants` runs over `node_modules/typescript/lib` on every PR, so the codebase-only contract is regression-gated on real dense code, not just synthetic fixtures. With S1–S4 done, the S7 real-repo corpus gate landed, and the GA recommendation finalized for the internal-tool bar (CAS ADR 0034 §4), every autonomous production-validation item is complete. CI is unblocked + reliably green (the parallel-worker Roslyn-sidecar build race was fixed — workers reuse a pre-built dll). **371 tests passing**;
build/typecheck pass. Design is recorded in CAS Decisions 0001–0035. See
[`architecture.md`](./architecture.md), [`backlog.md`](./backlog.md),
[`pathfinding-and-graph-index.md`](./pathfinding-and-graph-index.md), [`csharp-roslyn-provider.md`](./csharp-roslyn-provider.md), and [`evaluation-harness.md`](./evaluation-harness.md).
